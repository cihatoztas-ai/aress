GÜNCELLENDİ▼
Her sayfa için 3 katman uygulanır. Hiçbiri atlanamaz.
Katman 1 — Statik HTML (data-i18n)
// Tüm statik metin elementlerine zorunlu
<button data-i18n="sp_add_btn">+ Ekle</button>
<label data-i18n="th_material">Malzeme</label>
<input data-i18n-placeholder="sp_note_placeholder">
Katman 2 — JS Render (tv() helper)
// Her sayfanın script bloğuna ekle — esc()'den hemen sonra
function tv(key, tr) {
  var v = window.t ? window.t(key) : key;
  return (v === key) ? tr : v;
}

// Render fonksiyonlarında kullanım:
tv('sp_no_cuts', 'Henüz kesim eklenmemiş')
tv('sp_cut_done', 'Kesildi')
Katman 3 — _onLangChange
// Tüm render fonksiyonlarını çağırmalı — sadece yukle() yetmez
window._onLangChange = function() {
  yukle(); renderTracker(); renderFotolar();
  renderKesimler(); renderBukumler(); renderMarkalar();
  renderTestler(); renderMalzeme(); renderNotlar();
  renderLog(); renderBelgeler(); renderQR();
};

K-01 JS Syntax Kontrolü — Her Sayfada Zorunlu
▼
Her sayfa tesliminden önce Node.js ile syntax kontrolü yapılır. Tarayıcıda test öncesi bu adım atlanamaz. Hata varsa teslim kabul edilmez.
Kontrol Komutu (terminal)
node -e "
  var fs=require('fs');
  var c=fs.readFileSync('sayfa.html','utf8');
  var s=c.indexOf("'use strict';"), e=c.lastIndexOf('</script>');
  try{new Function(c.slice(s,e));console.log('✅ JS OK');}
  catch(err){console.log('❌ HATA:',err.message);}
"
Tüm Sayfaları Toplu Tara
# app/ klasöründen çalıştır
for f in *.html; do
  node -e "var fs=require('fs'),c=fs.readFileSync('$f','utf8'),s=c.indexOf("'use strict';"),e=c.lastIndexOf('</script>');if(s<0){console.log('⏭ JS yok: $f');process.exit(0);}try{new Function(c.slice(s,e));console.log('✅ $f');}catch(err){console.log('❌ $f: '+err.message);}"
done
⚠️ Sık karşılaşılan hatalar:
• JS string içinde tek tırnak → font-family:'Barlow' gibi CSS kaldır veya kaçır
• Python triple-quote içinde ' → çıktıda ' olur, JS string kırılır
• QRCode, Swiper gibi CDN kütüphaneleri yüklenmeden çağrılırsa → ReferenceError

R-02 Büyük Değişiklik Protokolü
YENİ▼
Bir sayfada 3+ farklı kural kategorisi değiştirilecekse, önce değişiklikler parçalara bölünür ve liste onaylanır. Sonra parça parça uygulanır. Her parça kendi K-01 syntax kontrolünü geçmeden bir sonrakine geçilmez.
İş Akışı
Başlamadan önce → "Bu sayfada ne yapacağız?" sorusunu sor. Değişiklikleri parçalara böl, her parçayı listele, onay al.
Her parça sonunda → K-01 syntax kontrolü çalıştır. ✅ geçtikten sonra dosyayı çıkar, bir sonraki parçaya geç.
Parça boyutu → CSS / Layout · Dil / D-01 · Güvenlik / G-01 · İşlevsellik (P-01, T-01 vb.) — her biri ayrı parça.
💡 Bu kural sayesinde: eksik kalan şeyler azalır, hata ayıklaması kolaylaşır, her parça test edilebilir hale gelir.
F-02 Minimum Font Boyutu
YENİ▼
Kullanıcının okuması gereken hiçbir metin 12px'in altına inemez. Bu kural tüm sayfalarda zorunludur.
✅ İzin Verilen
Tablo td → min 12px
Tablo th → min 12px
Badge / mini-badge → 12px
Stat kart label → 11-12px
Meta / tarih bilgisi → 12px
❌ Yasak
font-size: 9px
font-size: 10px
font-size: 11px (okunabilir metinlerde)
İstisna: QR kod içi, print CSS
💡 Kontrol: Sayfa tesliminden önce grep "font-size:[89]px\|font-size:1[01]px" sayfa.html ile tara. Aşağıdaki istisnalar hariç tut:
• QR kod içi — sınırlı alan, farklı ortam
• Print / PDF CSS — @media print veya inline PDF HTML string'leri
• Sort ok işaretleri (↕ ↑ ↓) — 9px olabilir, sembol, okunacak metin değil
R-03 Program Bütünlüğü — Çapraz Sayfa Kontrolü
YENİ▼
Bir sayfada yeni teknik standart veya kural oluşturulduğunda iki şey yapılır: (1) O andan itibaren yapılan tüm yeni sayfalarda standart anında ve zorunlu olarak uygulanır. (2) Daha önce tamamlanmış sayfaların şartname kartlarına "Bekleyen Değişiklikler" notu düşülür, sıraları gelince uygulanır. Böylece hiçbir şey atlanmaz ve program bütünlüğü korunur.
İş Akışı
Yeni standart oluşunca → "Bu kural hangi sayfalara uygulanmalı?" sorusunu sor. Yeni sayfalarda: anında uygula, bekleme.
Eski tamamlanmış sayfalar için → Şartname kartı "Bekleyen" bölümüne kural kodu + kısa açıklama ekle. R-01 ile sırası gelince uygulanır.
Yeni sayfa çalışması başlarken → Kurallar listesini baştan tara — bellekte olmayan kural olabilir. Şartname kartını oluştur ve uygulanan kuralları yaz.
✅ Doğru
F-02 kuralı çıktı
→ Devam eden devreler.html'e anında uygulandı
→ spool_detay bekleyenlerine eklendi
→ bukum bekleyenlerine eklendi
❌ Yanlış
F-02 kuralı çıktı
→ Sadece eski sayfaların bekleyenine eklendi
→ devreler.html'de atlandı
→ Sonra geri dönmek gerekti
💡 Kural belliyse bekleme yoktur. Yeni sayfada atlanan her kural = ileride geri dönme maliyeti. Supabase geçişinde özellikle kritik.
